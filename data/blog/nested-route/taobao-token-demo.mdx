## 📋 淘口令自动复制演示

本页面演示了如何在微信环境中自动复制淘口令并弹出友好提示。

### 使用场景

当用户在微信中打开商品链接时：
1. 🔍 **自动识别** - 检测 User-Agent 判断是否在微信中
2. 📋 **自动复制** - 将淘口令复制到剪贴板
3. 🔔 **弹窗提示** - 显示复制成功并提供引导

---

## 🎯 基础用法

在 MDX 文件中直接使用：

```mdx
import { TaoBaoTokenPopup } from '@/components/MDXComponents'

<TaoBaoTokenPopup 
  token="CZ0001 ABC1234"
  title="口令已复制"
  message="请打开手机淘宝查看宝贝"
/>
```

### 实际效果

{/* 在微信中访问此页面会自动弹出 */}
<TaoBaoTokenPopup 
  token="CZ8848 TB666666"
  title="🎉 淘口令已自动复制"
  message="请切换至【手机淘宝】APP 即可查看宝贝详情"
/>

## ⚙️ 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `token` | string | "CZ0001 ABC1234" | 淘口令内容 |
| `title` | string | "淘口令已复制" | 弹窗标题 |
| `message` | string | 自定义提示 | 提示信息 |
| `taobaoLink` | string | - | 淘宝商品链接（可选） |
| `autoShow` | boolean | true | 是否自动显示 |
| `delay` | number | 500 | 延迟显示时间（毫秒） |

## 💡 实现原理

### 环境检测

```typescript
const ua = window.navigator.userAgent.toLowerCase()
const isWX = ua.indexOf('micromessenger') !== -1
```

### 复制逻辑

```typescript
// 优先使用现代 Clipboard API
if (navigator.clipboard && navigator.clipboard.writeText) {
  await navigator.clipboard.writeText(token)
} else {
  // 兼容旧版浏览器
  const textArea = document.createElement('textarea')
  textArea.value = token
  document.body.appendChild(textArea)
  textArea.select()
  document.execCommand('copy')
  document.body.removeChild(textArea)
}
```

### 样式设计

- 使用 `fixed` 定位覆盖全屏
- `zIndex: 9999` 确保在最上层
- 半透明遮罩 + 毛玻璃效果
- 响应式设计适配不同屏幕

### 1. 浏览器兼容性

- 提供明确的提示信息
- 允许用户关闭弹窗

### 3. 性能优化

```typescript
// 使用防抖避免频繁触发
useEffect(() => {
  const timer = setTimeout(() => {
    handleAutoCopy()
  }, delay)
  
  return () => clearTimeout(timer)
}, [])
```

## 💬 常见问题

### Q: 为什么有时不会自动弹出？

A: 组件会检测是否在微信环境中。如果不是微信，不会显示弹窗。

### Q: 可以禁用自动复制吗？

A: 设置 `autoShow={false}`，然后通过其他方式手动触发。

### Q: 如何修改默认样式？

A: 可以直接修改组件源码中的 `styles` 对象，或传入自定义样式。
