## 🧪 Middleware 跳转测试

本页面用于验证 `/go/` 路由的跳转功能是否正常工作。

### 测试方法

点击以下链接，应该会自动重定向到对应的电商平台：

#### 桌面端测试（预期跳转京东）

- [测试 iPhone 15 Pro Max](/go/iphone15-pro-max) - 应跳转到京东
- [测试华为 Mate 60 Pro](/go/mate60-pro) - 应跳转到淘宝
- [测试戴森 V15](/go/dyson-v15) - 应跳转到京东

#### 移动端测试（预期跳转淘宝）

在手机上打开或使用浏览器的移动设备模式：

- [测试 iPhone 15 Pro Max](/go/iphone15-pro-max) - 应跳转到淘宝
- [测试华为 Mate 60 Pro](/go/mate60-pro) - 应跳转到淘宝
- [测试戴森 V15](/go/dyson-v15) - 应跳转到淘宝

### 技术说明

```typescript
// middleware.ts 工作位置
请求 → /go/iphone15-pro-max
       ↓
Middleware (Edge Function)
       ↓
识别 User-Agent
       ↓
PC → 京东 | Mobile → 淘宝
       ↓
307 临时重定向
```

### 预期结果

✅ **成功标志**：
- 点击链接后立即跳转（~10ms）
- 无需加载中间页面
- URL 变为电商平台链接

❌ **失败标志**：
- 页面显示 404
- 长时间加载后无响应
- 跳转到错误的平台

### 调试信息

如果跳转失败，请检查：

1. `middleware.ts` 是否正确编译
2. `.next/server/middleware.js` 是否存在
3. 控制台是否有错误日志

查看 Middleware 日志：

```bash
# 开发环境
tail -f .next/server/middleware.js.map

# 生产环境
vercel logs --follow
```
